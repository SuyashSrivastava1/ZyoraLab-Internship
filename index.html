The 15-Day Roadmap

Follow my technical journey through the 3-week foundation, build, and shipping phases. Click any day below to reveal detailed learning topics in native HTML:

  1. DAY 01: Kickoff + Developer Workflow (Week 1 — Foundations)

    Setting up a professional environment is the foundation of high-velocity development.

    • Git Setup & SSH Configuration
    • Advanced terminal customizing & shell aliases
    • VS Code / IDE environment enhancements
    • Branching strategies and PR workflows
  2. DAY 02: HTML Deep-Dive & Semantic Multi-Section Architecture (Week 1 — Foundations)

    Understanding semantic markup and structuring documents for maximum accessibility, SEO, and native browser features without reliance on CSS/JS.

    • HTML5 Semantic Landmarks (<header>, <nav>, <main>, <section>, <article>, <footer>)
    • WAI-ARIA roles, landmark headings, and explicit screen reader optimizations
    • Native zero-JS accordions using <details> and <summary>
    • Accessible data structures (<table> with captions and scope attributes) and forms with explicit labels
  3. DAY 03: CSS I — Box Model & Flexbox (Week 1 — Foundations)

    Mastering layout alignment and understanding the mechanics of CSS rendering.

    • Margin collapsing & box sizing
    • Flexbox alignment axes & wrapping patterns
    • Growth & Shrink coefficients (flex-grow, flex-shrink)
    • Grid overlay structures & flex styling pairings
  4. DAY 04: CSS II — Responsive Design (Week 1 — Foundations)

    Building designs that dynamically scale and align perfectly across all screen devices.

    • Mobile-first media queries and breakpoints
    • CSS clamp(), min(), and max() fluid functions
    • Grid Areas, auto-fill, and auto-fit rules
    • Subgrid features & SVG scaling logic
  5. DAY 05: JavaScript I — DOM & Events (Week 1 — Foundations)

    Manipulating page structures dynamically and wiring up form validation and event listeners.

    • Node queries, insertions, and removal APIs
    • Event propagation: capture, bubble, and preventDefault
    • Event delegation strategies for dynamic lists
    • Form input handling and live feedback
  6. DAY 06: JavaScript II — Arrays, Objects & APIs (Week 2 — JavaScript & React)

    Managing data transformations, asynchronous actions, and external network interactions.

    • Functional array operations (map, filter, reduce)
    • Destructuring, rest parameters, and spread operations
    • Asynchronous promises, async/await, and fetch call cycles
    • Parsing & validating API inputs
  7. DAY 07: React I — Components & Props (Week 2 — JavaScript & React)

    Transitioning to declarative components and thinking in structured layout hierarchies.

    • Understanding the Virtual DOM and JSX compile processes
    • Props passing, component structuring, and re-usability
    • Keys, list rendering, and fragment usage
    • Conditional rendering patterns
  8. DAY 08: React II — State & Hooks (Week 2 — JavaScript & React)

    Creating interactive components using React hooks and understanding state lifecycle cycles.

    • useState: triggering re-renders & handling complex state objects
    • useEffect: handling side-effects, subscriptions, and cleanups
    • Rules of hooks & common mistakes
    • Controlled components & form interactions
  9. DAY 09: 🚀 PROJECT KICKOFF (Week 2 — Project Kickoff)

    Planning architecture, structuring source code templates, and aligning on product scope.

    • Project wireframing and design layout validation
    • Boilerplate setup (Vite + React + CSS setup)
    • Folder architectures (components, hooks, pages, services)
    • GitHub Issue tracking & project boards setup
  10. DAY 10: Team Build Day (Week 2 — Project Kickoff)

    Pair programming, combining individual components into a cohesive core, and resolving conflicts.

    • Merging feature branches & git conflict resolution
    • Interface integrations and local state validation
    • Conducting team review sessions and code checks
    • Shared utility and helper setup
  11. DAY 11: Mid-Project Review (Week 3 — Build, Ship & Demo)

    Gathering feedback on early beta builds, prioritizing adjustments, and verifying progress.

    • UI audits and interactive usability checkups
    • Analyzing performance bottlenecks or rendering delays
    • Pruning excessive scopes to guarantee deployment timing
    • Staging environment deployments
  12. DAY 12: Deployment & Git Collaboration (Week 3 — Build, Ship & Demo)

    Deploying product structures to production hosts and verifying automation pipelines.

    • Deploying on Vercel, Netlify, or GitHub Pages
    • Configuring environment variables & API limits
    • CI/CD setup with GitHub Actions
    • Checking security headers & HTTPS certificate bindings
  13. DAY 13: Bug-fix & Polish (Week 3 — Build, Ship & Demo)

    Addressing edge-cases, stabilizing layout shifts, and polishing core micro-animations.

    • Core Web Vitals optimizations (LCP, CLS, FID)
    • Cross-device, cross-browser layout fixes
    • Custom animations and interface transitions
    • Code splitting, asset optimizations, and lazy loading
  14. DAY 14: Final Submission (Week 3 — Build, Ship & Demo)

    Finalizing code, packaging instructions, and preparing user documentation.

    • Writing README guides, feature summaries, and user manuals
    • Final testing on main release branches
    • Asset packaging and release tagging
    • Pre-flight deployment checklist confirmation
  15. DAY 15: 🎓 DEMO DAY (Week 3 — Build, Ship & Demo)

    Showcasing the completed implementation to users and celebrating the learning path.

    • Deploying finalized builds to production live servers
    • Conducting project showcase presentations
    • Receiving peer feedback & future roadmap planning
    • Celebrating graduation & next milestones definition
